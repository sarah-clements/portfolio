Donation Platform




I integrated Stripe’s payment processing library client-side in order for We Vote to accept donations securely through their web app. The Checkout library provides a secure form for entering credit card details that transmits encrypted payment data directly to Stripe and returns a unique token. Once a token is returned for each transaction, it’s passed to our server. We Vote's server will then call Stripe’s API in order to create customer accounts (for storing payment transaction details), process one-time donations and to create recurring monthly donations. Additionally, I created datatables in order to log all transactions and save recurring donation plans created on the fly (we do not handle or store actual credit card numbers).

View the ReactJS code or the Django code on Github.

MY CONTRIBUTIONS

  • Collaborated with the Director of Engineering to architect the process end-to-end and to determine requirements for the Postgres datatables.
  • Integrated Stripe’s Checkout library (vanilla Javascript) with React.
  • Created custom features for the interface (I did not create the donation interface) that included custom amount and monthly donation options and a ‘processing your donation’ page that will route to either a thank-you page if successful or remain on the donation page along with an error message if the transaction fails.
  • Created an API endpoint in React (using Flux) to transmit the unique token and additional data to our server.
  • Created the view, controller and models in Django for posting data to the Stripe API in order to process transactions and to save pertinent information to our database (ex: If a customer wants to create a $30 monthly donation and that donation plan does not exist in our Stripe account, we’ll create it with Stripe, subscribe them and also save it to our database for future use).
  • Created a Django controller and model dict for error handling that generates dynamic messages based on internal server errors, stripe errors or specific credit card errors (incorrect pin, invalid card number, etc); this message is returned to the user in React if the donation fails.
  • Created a Readme for first-time contributors wanting to work on this feature

Select Previous Ballot Guides




We Vote’s web app allows users to create personal ballot guides for local and national upcoming elections based on their address. For this feature, we wanted to allow users to access any personal ballot guides from past elections that they had created. The gear icon to access this modal will only show if a user has created a ballot guide from a previous election.

View the ReactJS code or the Django code on Github.

MY CONTRIBUTIONS

  • Collaborated with the UX Designer to implement the modal’s interface design.
  • Created the React component and stylesheet (using SASS) for the modal pop-up (that is conditional on a user having a ballot guide from a previous election).
  • Created the API call and event listeners in React for retrieving the ballot guide list (previously saved to our database based on a unique user ID).
  • Created the view and controller functions for the API in our server (to retrieve the ballot guide list).
